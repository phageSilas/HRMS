## 1. 文档说明
### 1.1 文档目的
本文档基于 HRMS 人资管理系统的 PRD、全局系统分析说明书、后端总体系统分析说明书和前端总体系统分析说明书，对 9 个业务模块和公共系统能力进行测试分析，明确各模块的测试维度、测试要点、异常场景和质量关注点。

本文档的输出是后续**测试用例设计**和**测试执行**的输入依据，不替代测试用例。

### 1.2 测试分析方法论
本文档遵循如下测试分析方法论：

```mermaid
flowchart LR
    A["1. 理解产品"] --> B["2. 识别范围"]
    B --> C["3. 分析特点"]
    C --> D["4. 细化场景"]
    D --> E["5. 输出测试要点"]
```

+ **理解产品**：基于 PRD、系分文档理解业务目标和系统边界。
+ **识别范围**：确定被测模块的功能范围、接口范围和数据范围。
+ **分析特点**：从 7 个维度分析——功能、系统交互、数据、异常边界、非功能、安全、兼容性。
+ **细化场景**：将分析结论转化为可执行的测试场景。
+ **输出测试要点**：作为后续测试用例设计的基准。

#### 测试设计方法
在细化场景和输出测试要点时，依据以下测试设计方法，确保覆盖的充分性和用例的简洁性：

| 方法 | 适用场景 | 在本文档中的应用示例 |
| --- | --- | --- |
| **等价类划分** | 输入参数存在有效/无效取值区间 | 请假天数（有效：1~30天；无效：0/负数/31天+）；用户账号长度 |
| **边界值分析** | 存在明确数值或长度边界 | 分页 pageSize=1/10/100/1000；请假天数 ≤3 天与 >3 天的审批链切换点 |
| **判定表** | 多个条件组合决定不同结果 | 请假审批链（类型×天数组合 → 审批节点数）；薪资异常检测（异常类型×级别 → 处理方式） |
| **状态转换测试** | 业务对象有明确状态机 | 审批状态（待审批→审批中→通过/驳回/已撤回）；员工状态（在职→离职→冻结） |
| **成对组合测试（Pairwise）** | 多条件查询筛选的参数组合 | 员工列表多条件搜索（部门×状态×角色×关键字 → 减少组合数 90%+） |
| **错误推测法** | 经验驱动的典型易错场景 | 并发提交、重复打卡、上下游依赖不可用、超时断连 |

## 2. 测试策略
### 2.1 测试层级与维度
测试分析覆盖以下 7 个维度，对应不同的分析视角：

| 维度 | 分析视角 | 关注重点 |
| --- | --- | --- |
| 功能分析 | 业务需求 | 功能是否满足 PRD，流程是否完整，业务规则是否正确 |
| 系统交互分析 | 时序与接口 | 模块间调用关系、数据流转、接口参数与返回值 |
| 数据分析 | 数据模型 | 数据完整性、唯一约束、状态转换、数据迁移 |
| 异常与边界分析 | 容错能力 | 输入边界、中断恢复、并发冲突、超时处理 |
| 非功能分析 | 性能与体验 | 响应时间、并发能力、页面加载、大列表渲染 |
| 安全分析 | 权限与数据安全 | 认证绕过、越权访问、敏感数据泄露、XSS/CSRF |
| 兼容性分析 | 环境适配 | 浏览器兼容、分辨率适配、不同角色的差异视图 |


### 2.2 功能测试策略
功能测试以模块为单元，覆盖以下内容：

+ **正向流程**：用户按预期路径操作，验证业务状态正确流转。
+ **分支流程**：不同条件分支（角色、状态、数据类型）下的页面展示和业务行为。
+ **反向流程**：输入异常、操作顺序异常、权限不足等情况下的提示和兜底。

各模块的功能测试基于系分文档中定义的**用例图**和**用例描述表**展开，确保 UC-XX 级别的用例被 100% 覆盖。

### 2.3 接口与集成测试策略
接口测试基于后端系分的 REST API 接口清单，覆盖：

+ **参数校验**：必填参数缺失、参数类型错误、参数长度超限、枚举值越界。
+ **业务逻辑**：同一接口在不同业务状态下的返回值差异。
+ **异常码**：后端定义的全部 19 个异常码（20000~50002）是否触发并正确返回。
+ **数据权限**：同一接口被不同角色调用返回的数据范围是否正确。
+ **SSE 流式接口**：AI 对话接口的 SSE 事件顺序、断连恢复、超时处理。

### 2.4 数据测试策略
数据测试关注：

+ **数据完整性**：创建/更新操作后，数据库各字段值是否正确写入。
+ **字段约束**：唯一索引（如用户账号、工号）、非空约束、默认值、枚举值。
+ **状态转换**：审批状态、员工状态、薪资批次状态等状态机的合法路径和非法路径。
+ **数据归档**：归档任务是否正确迁移历史数据，归档后的查询是否正常。
+ **乐观锁**：并发更新时 `version` 字段递增和冲突检测是否正确。

### 2.5 异常与边界测试策略
| 异常类型 | 测试场景 |
| --- | --- |
| 输入边界 | 文本字段最大长度、数字字段最大值/最小值、日期范围越界 |
| 网络异常 | 请求超时、连接断开、请求重试、幂等性 |
| 并发冲突 | 多人同时编辑同一条数据、重复提交审批、同一设备多地打卡 |
| 状态冲突 | 已审批的申请再操作、已离职员工再发起审批、已发放的批次再计算 |
| 依赖异常 | 下游服务不可用、Redis 断连、LLM 超时降级 |


### 2.6 非功能测试策略
| 维度 | 指标 | 测试方法 |
| --- | --- | --- |
| 响应时间 | API < 500ms（读）、< 2s（写） | 性能测试工具压测 |
| 并发能力 | 核心接口支持 200 并发 | 逐步加压测试 |
| 页面加载 | 首屏 < 2s、列表页 < 1.5s | Lighthouse / 埋点 |
| AI 对话 | 首 token < 5s、完整回复 < 15s | SSE 响应计时 |
| 大数据量 | 10 万级员工列表查询、百万级考勤记录 | 分页查询测试、虚拟滚动验证 |


### 2.7 安全测试策略
| 安全维度 | 测试要点 |
| --- | --- |
| 认证 | Token 有效期、Token 伪造、未登录访问受保护接口、刷新 Token 机制 |
| 越权 | 普通员工调用 HR 接口、跨部门查看数据、修改他人数据 |
| 数据脱敏 | 手机号、身份证号、银行卡号、薪资字段的脱敏展示 |
| XSS | 用户输入的富文本、AI 回复的 Markdown 渲染是否安全过滤 |
| CSRF | 敏感操作是否需要验证等 |
| 敏感数据 | 工资条明细是否被前端持久化、离开页面是否清理 |


### 2.8 兼容性测试策略
| 维度 | 范围 |
| --- | --- |
| 浏览器 | Chrome / Edge 最新两个大版本 |
| 分辨率 | 1920×1080（主流）、1440×900、1366×768 |
| 角色视图 | 系统管理员、HR、部门主管、财务、普通员工五种角色的页面差异 |

| --- | --- | --- |
| UC-06-01 | 登录认证 | 用户名密码正确/错误登录、空输入校验、登录成功后的 Token 保存和跳转 |
| UC-06-02 | 用户管理 | 用户 CRUD、用户名唯一性校验、状态启用/停用、密码重置 |

| UC-06-04 | 菜单管理 | 菜单树维护、目录/菜单/按钮类型区分、路由与权限标识绑定 |
| UC-06-05 | 权限分配 | 角色菜单树勾选、保存后权限即时生效、权限变更后用户重新登录 |


#### 接口测试要点
| 接口 | 测试关注点 |
| --- | --- |
| `POST /api/v1/auth/login` | 用户名/密码校验、错误次数锁定、返回体中的 Token 和用户信息 |

+ 用户账号、角色编码的唯一索引校验。
+ 系统管理员不展示菜单管理中的无权限选项。
+ 部门主管的角色数据范围限制为本部门。

| UC-09-04 | 离职管理 | 离职原因填写、交接人配置、离职日期计算、离职后账号冻结 |


#### 接口测试要点
| 接口 | 测试关注点 |
| --- | --- |
| `POST /api/v1/entry-applications` | 候选人信息完整性校验、草稿与正式提交的状态区别 |
| `POST /api/v1/entry-applications/{id}/confirm` | 确认入职后的员工创建、入职日期与合同起期的联动 |
| `GET /api/v1/transfer-applications` | 角色数据权限过滤、状态筛选 |


#### 异常场景
+ 入职申请提交审批后再次修改。
+ 转正评估中薪资调整超出预设范围。
+ 调岗生效日期早于当前日期。
+ 离职日期与合同到期日的冲突处理。

#### 数据与权限校验
+ 主管只能查看本部门的转正和调岗申请。
+ 离职后账号登录失败，但历史数据可查询。
+ 入职确认后自动生成员工数据和合同数据。

#### 调岗审批链测试矩阵
| 审批步骤 | 审批人 | 测试场景 |
| --- | --- | --- |
| 1. 原部门审批 | 原部门负责人 | 同意/驳回/转交操作、超时未处理、原部门负责人与申请人同一人（自动跳过？） |
| 2. 新部门审批 | 新部门负责人 | 同意/驳回操作、新部门负责人与原部门负责人为同一人（自动跳过？） |
| 3. HR 备案 | HR 专员 | 备案后自动更新员工部门/岗位字段、生效日期落地 |


**边界场景：**

#### 薪资异常检测规则测试矩阵
| 异常规则 | 级别 | 触发条件 | 测试场景 |
| --- | :---: | --- | --- |
| 请假天数 > 15 天 | YELLOW | 员工当月请假天数 > 15 | HR 确认后可继续核算、确认后异常标记消除 |
| 加班时长 > 50 小时 | YELLOW | 加班时长 > 50 小时 | HR 确认后可继续、确认操作记录审计日志 |
| 较上月变动 > 30% | RED | 实发金额较上月涨跌幅 > 30% | 需人工确认原因、确认后降级为 YELLOW 可继续 |
| 未设置薪资档案 | BLOCK | 员工无薪资档案 | 补齐档案后重新触发异常检测、BLOCK 期间提交审批按钮禁用 |

**边界与异常场景：**

+ 多项异常同时命中同一员工 → 取最高级别异常展示（BLOCK > RED > YELLOW）。
+ HR 确认 YELLOW 异常后重新计算，异常列表更新。
+ BLOCK 异常全部解除前提交审批按钮保持禁用。
+ 异常检测的精度：金额比较采用 Decimal 类型，避免浮点精度误差。
+ 异常处理操作记录计入审计日志。

| 1 | 员工向 AI 提问"帮我请明天年假" | M9 AI 智能助手 | 意图识别准确、参数提取（类型/日期）正确 |

| 3 | 员工点击后跳转请假页 | M9 → M3 考勤管理 | 页面参数传递、AI 预填字段 |
| 4 | 员工提交请假 | M3 → M7 审批中心 | 审批流正常触发、与场景三请假规则一致 |
| 5 | AI 追问"显示我的年假余额" | M9 → M1 员工档案 | RAG 检索 + 数据查询权限校验 |


**典型缺陷场景：** AI 错误理解意图发起错误操作、跳转页面后 AI 预填字段与实际表单字段不一致、跨模块数据查询的权限越界。

## 7. 风险与应对
| 风险 | 影响 | 应对策略 |
| --- | --- | --- |
| 接口文档与接口实现不一致 | 接口测试阻塞 | 联调阶段同步更新接口文档，接口测试用例与后端联调同步推进 |
| 数据权限逻辑复杂 | 越权漏洞 | 设计专门的越权测试用例，覆盖每组角色权限组合 |
| AI SSE 流式交互不稳定 | AI 对话测试不充分 | 覆盖断连、超时、空回复、异常回复等异常场景 |
| 涉薪计算精度问题 | 金额错误 | 设计多组核算数据对比测试，验证四舍五入和截断规则 |
| 浏览器兼容性问题 | 页面错乱 | 在 Chrome / Edge / Firefox 上执行核心页面兼容性测试 |
| 长时间未归档的数据量过大 | 查询/归档性能问题 | 大数据量环境下的分页查询和归档任务性能验证 |
