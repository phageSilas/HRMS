# HRMS 项目启动验证指南

## 验证前准备

### 1. 环境要求

| 环境 | 版本要求 | 检查命令 |
|------|----------|----------|
| JDK | 17+ | `java -version` |
| Maven | 3.6+ | `mvn -version` |
| MySQL | 8.0+ | `mysql --version` |
| Redis | 6.0+ | `redis-cli --version` |

### 2. 数据库准备

```bash
# 1. 登录MySQL
mysql -u root -p

# 2. 执行建表脚本
source backend/sql/schema/00_init_database.sql

# 或者逐个执行
source backend/sql/schema/sys_user.sql
source backend/sql/schema/sys_role.sql
# ... 其他表

# 3. 执行初始化数据
source backend/sql/data/init_dict.sql
source backend/sql/data/init_admin.sql
```

## 验证步骤

### 步骤1：配置数据源

编辑 `backend/hrms-server/src/main/resources/application-dev.yaml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/hrms?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: root
    password: 你的密码
```

### 步骤2：启动Redis

```bash
# Windows
redis-server

# Linux/Mac
redis-server /etc/redis/redis.conf
```

### 步骤3：编译项目

```bash
cd backend
mvn clean compile -DskipTests
```

**预期结果**：
```
[INFO] BUILD SUCCESS
```

### 步骤4：运行测试

```bash
mvn test
```

**注意**：现有测试用例有问题，需要修复测试期望值（code应为0而非200）。

### 步骤5：启动应用

```bash
cd hrms-server
mvn spring-boot:run
```

**预期结果**：
```
Started HrmsServerApplication in X.XXX seconds
```

### 步骤6：验证接口

```bash
# 健康检查
curl http://localhost:8080/actuator/health

# 测试接口
curl http://localhost:8080/api/system/summary
```

**预期返回**：
```json
{
  "code": 0,
  "message": "操作成功",
  "data": "system module: permission, organization, employee archive"
}
```

## 常见问题

### 1. 端口被占用

**错误**：`Port 8080 already in use`

**解决**：
```yaml
# application.yaml
server:
  port: 8081
```

### 2. 数据库连接失败

**错误**：`Communications link failure`

**解决**：
- 检查MySQL是否启动
- 检查连接地址、用户名、密码
- 检查防火墙设置

### 3. Redis连接失败

**错误**：`Unable to connect to Redis`

**解决**：
- 检查Redis是否启动
- 检查Redis配置（host、port）

### 4. RabbitMQ连接失败

**错误**：`Failed to connect to RabbitMQ`

**解决**：
- 安装并启动RabbitMQ
- 或临时禁用RabbitMQ自动配置：
```java
@SpringBootApplication(exclude = {
    org.springframework.boot.autoconfigure.amqp.RabbitAutoConfiguration.class
})
```
