## 五、接口规范

> 与后端统一返回体 `Result<T>` 保持一致。

### 5.1 统一返回体

**后端响应格式**：
```json
{
  "code": 0,
  "message": "success",
  "data": { ... }
}
```

**前端类型定义**：
```typescript
// types/api.ts

/**
 * 统一返回体（对应后端 Result<T>）
 */
export interface Result<T> {
  code: number;
  message: string;
  data: T;
}

/**
 * 分页返回体（对应后端 PageResult<T>）
 */
export interface PageResult<T> {
  records: T[];
  total: number;
  pageNum: number;
  pageSize: number;
}
```

### 5.2 分页参数

| 参数 | 类型 | 说明 |
|------|------|------|
| pageNum | number | 页码（从 1 开始） |
| pageSize | number | 每页大小 |
| total | number | 总记录数（响应） |
| records | T[] | 数据列表（响应） |

**示例**：
```typescript
// 查询参数
interface PageQuery {
  pageNum: number;
  pageSize: number;
}

// 分页请求
export async function getEmployeeList(params: PageQuery & EmployeeQuery) {
  return request<Result<PageResult<EmployeeListVO>>>('/employees', {
    method: 'GET',
    params,
  });
}
```

### 5.3 错误码处理

| 码段 | 类型 | 前端处理 |
|------|------|----------|
| 0 | 成功 | 正常返回 data |
| 40001-40099 | 参数错误 | 提示参数错误 |
| 40100-40199 | 认证授权错误 | 跳转登录页 |
| 50001-50099 | 系统内部错误 | 提示系统繁忙 |
| 60001-60599 | 业务错误 | 显示业务错误信息 |

## 六、请求封装规范

### 6.1 baseURL 配置

```typescript
// utils/request/index.ts

import axios from 'axios';

const request = axios.create({
  baseURL: 'http://localhost:8080',  // 后端地址
  timeout: 10000,
  headers: {
    'Content-Type': 'application/json',
  },
});

export default request;
```

### 6.2 Token 注入

```typescript
// 请求拦截器：注入 Token
request.interceptors.request.use(
  (config) => {
    const token = localStorage.getItem('token');
    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
  },
  (error) => {
    return Promise.reject(error);
  }
);
```

### 6.3 响应拦截

```typescript
// 响应拦截器：处理 Result<T> 格式
request.interceptors.response.use(
  (response) => {
    const { code, message, data } = response.data;
    
    // 成功
    if (code === 0) {
      return data;
    }
    
    // 认证失败
    if (code >= 40100 && code <= 40199) {
      localStorage.removeItem('token');
      window.location.href = '/login';
      return Promise.reject(new Error(message));
    }
    
    // 其他错误
    return Promise.reject(new Error(message));
  },
  (error) => {
    // 网络错误等
    return Promise.reject(error);
  }
);
```

### 6.4 services 文件模板

```typescript
// services/employee/index.ts

import request from '@/utils/request';
import type { Result, PageResult } from '@/types/api';
import type { Employee, EmployeeCreateRequest, EmployeeQuery } from '@/types/employee';

/**
 * 获取员工列表
 */
export async function getEmployeeList(params: EmployeeQuery) {
  return request.get<Result<PageResult<Employee>>>('/employees', { params });
}

/**
 * 获取员工详情
 */
export async function getEmployeeDetail(id: number) {
  return request.get<Result<Employee>>(`/employees/${id}`);
}

/**
 * 创建员工
 */
export async function createEmployee(data: EmployeeCreateRequest) {
  return request.post<Result<Employee>>('/employees', data);
}

/**
 * 更新员工
 */
export async function updateEmployee(id: number, data: Partial<EmployeeCreateRequest>) {
  return request.put<Result<Employee>>(`/employees/${id}`, data);
}

/**
 * 删除员工
 */
export async function deleteEmployee(id: number) {
  return request.delete<Result<void>>(`/employees/${id}`);
}
```

## 七、开发约定

### 7.1 页面开发规范

- **页面代码优先写在 `pages/` 目录**
- **页面只负责 UI 渲染和事件处理**
- **复杂逻辑抽离到 `hooks/` 或 `utils/`**

### 7.2 接口调用规范

- **统一从 `services/` 发起请求，禁止在页面中直接写 fetch/axios**
- **接口路径不要硬编码，统一在 services 文件中管理**

### 7.3 组件复用规范

- **通用组件放 `components/common/`**（如表格封装、弹窗）
- **业务组件放 `components/business/`**（如员工选择器、部门树选择器）

### 7.4 状态管理规范

- **页面私有状态留在页面内部**
- **跨页面共享状态才放 `models/`**

### 7.5 Git 提交规范

与后端保持一致，采用 Conventional Commits 格式：

```
<type>(<scope>): <subject>
```

**类型**：feat、fix、refactor、docs、test、chore

### 8.5 权限控制实现

#### 8.3.1 权限控制流程

```
用户登录
    │
    ▼
获取用户信息 + 权限列表
    │
    │  POST /auth/login
    │  Response: { token, userId, username, roleCode, permissions: [...] }
    │
    ▼
存储到 Umi initialState
    │
    │  app.ts -> getInitialState()
    │
    ▼
access.ts 定义权限规则
    │
    │  export default function access(initialState) {
    │    return {
    │      systemMenu: initialState.permissions.includes('system'),
    │      employeeMenu: initialState.permissions.includes('employee'),
    │      // ...
    │    };
    │  }
    │
    ▼
路由配置关联权限
    │
    │  .umirc.ts -> routes[].access
    │
    ▼
侧边栏菜单根据权限显示/隐藏
```

#### 8.3.2 权限标识规范

沿用后端 `sys_menu.perms` 字段格式：

```
模块:资源:操作

- system              → 系统管理模块
- system:user         → 用户管理
- system:user:list    → 查看用户列表
- system:user:create  → 创建用户
- employee            → 员工档案模块
- employee:list       → 查看员工列表
- approval            → 审批中心模块
- profile             → 个人中心模块
```
