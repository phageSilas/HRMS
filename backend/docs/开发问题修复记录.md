## 一、编译错误修复

### 1. UserMapper 缺少方法

**问题描述**：
```
找不到符号: 方法 selectPermissionsByUserId(java.lang.Long)
找不到符号: 方法 selectRolesByUserId(java.lang.Long)
```

**根本原因**：
`UserMapper` 接口未定义 `selectRolesByUserId` 和 `selectPermissionsByUserId` 方法，但 `AuthServiceImpl` 和 `UserServiceImpl` 中调用了这些方法。

**修复方案**：
在 `UserMapper.java` 中添加方法定义：

```java
/**
 * 根据用户 ID 查询角色列表。
 */
@Select("SELECT r.* FROM sys_role r " +
        "INNER JOIN sys_user_role ur ON r.id = ur.role_id " +
        "WHERE ur.user_id = #{userId} AND r.is_deleted = 0")
List<RoleDO> selectRolesByUserId(@Param("userId") Long userId);

/**
 * 根据用户 ID 查询权限列表。
 */
@Select("SELECT DISTINCT m.permission FROM sys_menu m " +
        "INNER JOIN sys_role_menu rm ON m.id = rm.menu_id " +
        "INNER JOIN sys_user_role ur ON rm.role_id = ur.role_id " +
        "WHERE ur.user_id = #{userId} AND m.is_deleted = 0 AND m.permission IS NOT NULL")
List<String> selectPermissionsByUserId(@Param("userId") Long userId);
```

**修复文件**：
- `backend/hrms-system/src/main/java/com/hrms/system/auth/mapper/UserMapper.java`

### 2. UserContext 缺少 getUsername() 方法

**问题描述**：
```
找不到符号: 方法 getUsername()
位置: 类型为com.hrms.common.security.UserContext的变量 userContext
```

**根本原因**：
`OperationLogAspect` 中调用了 `userContext.getUsername()`，但 `UserContext` 类未定义该字段和方法。

**修复方案**：
在 `UserContext` 类中添加 `username` 字段和 `getUsername()` 方法：

```java
public class UserContext {
    private final Long userId;
    private final Long deptId;
    private final String username;  // 新增
    private final List<Long> roleIds;

    public UserContext(Long userId, Long deptId, String username, List<Long> roleIds) {
        this.userId = userId;
        this.deptId = deptId;
        this.username = username;
        this.roleIds = roleIds != null ? roleIds : List.of();
    }

    // 兼容旧构造函数
    public UserContext(Long userId, Long deptId, List<Long> roleIds) {
        this(userId, deptId, null, roleIds);
    }

    public String getUsername() {
        return username;
    }

    // 其他 getter 方法...
}
```

**修复文件**：
- `backend/hrms-common/src/main/java/com/hrms/common/security/UserContext.java`

### 5. Servlet API 版本问题

**问题描述**：

## 二、配置错误修复

### 6. application.yaml 重复键

**问题描述**：
```
found duplicate key spring
 in 'reader', line 57, column 1:
    spring:
```

**根本原因**：
`application.yaml` 中有两个 `spring:` 键（第 1 行和第 57 行），违反 YAML 语法规范。

**修复方案**：
合并两个 `spring:` 块，将文件上传配置移到第一个 `spring:` 块内：

```yaml
spring:
  application:
    name: hrms-server
  
  # ... 其他配置 ...
  
  # 文件上传配置
  servlet:
    multipart:
      enabled: true
      max-file-size: 10MB
      max-request-size: 50MB

# 其他配置（非 spring 块）
mybatis-plus:
  # ...
```

**修复文件**：
- `backend/hrms-server/src/main/resources/application.yaml`

### 8. 数据库表名映射错误（重要）

**问题描述**：
```
Table 'hrms.user_d_o' doesn't exist
SQL: SELECT ... FROM user_d_o WHERE is_deleted=0 AND (username = ?)
```

**根本原因**：
MyBatis-Plus 默认将实体类名转换为下划线分隔的表名：
- `UserDO` → `user_d_o`（错误）
- 期望：`sys_user`（正确）

**修复方案**：
为所有实体类添加 `@TableName` 注解：

```java
@TableName("sys_user")
public class UserDO extends BaseEntity {
    // ...
}
```

**修复文件**（共 12 个实体类）：

   - 所有 `javax.servlet.*` 改为 `jakarta.servlet.*`
   - 其他 javax 包也需要检查

3. **配置文件需严格验证**
   - YAML 不允许重复键
   - 使用 IDE 插件检查语法错误

4. **统一导入路径**
   - 避免同一类在不同包中存在多个版本
   - 使用代码规范工具检查

## 七、运行环境问题修复（2026-07-10）

### 9. 数据库字段映射错误（登录功能）

**问题描述**：
登录接口返回错误，日志显示：
```
Unknown column 'success' in 'field list'
```

**根本原因**：
`LoginLogDO` 实体类字段名与数据库表 `sys_login_log` 字段名不匹配：

| 实体类字段 | 数据库字段 | 说明 |
|-----------|-----------|------|
| success | login_status | 登录结果 |
| ipAddress | login_ip | IP地址 |
| failReason | message | 失败原因 |
| location | login_location | 登录地点 |
| deviceInfo | browser + os | 设备信息 |

**修复方案**：
执行 ALTER TABLE 修改数据库字段名：

```sql
USE hrms;

-- 修改字段名以匹配实体类
ALTER TABLE sys_login_log
  CHANGE COLUMN login_status success TINYINT NOT NULL DEFAULT 1 COMMENT '登录结果(0失败1成功)';

ALTER TABLE sys_login_log
  CHANGE COLUMN login_ip ip_address VARCHAR(64) COMMENT '登录IP地址';

ALTER TABLE sys_login_log
  CHANGE COLUMN message fail_reason VARCHAR(255) COMMENT '失败原因';

ALTER TABLE sys_login_log
  CHANGE COLUMN login_location location VARCHAR(128) COMMENT '登录地点';

-- 添加缺失的字段
ALTER TABLE sys_login_log
  ADD COLUMN device_info VARCHAR(255) COMMENT '设备信息' AFTER location;

-- 合并 browser 和 os 数据到 device_info
UPDATE sys_login_log 
SET device_info = CONCAT(IFNULL(browser, ''), ' ', IFNULL(os, ''));
```

**修复状态**: ✅ 已修复

### 10. JWT密钥长度不足

**问题描述**：
登录时报错：
```
io.jsonwebtoken.security.WeakKeyException: The specified key byte array is 224 bits 
which is not secure enough for any JWT HMAC-SHA algorithm.
The JWT JWA Specification (RFC 7518, Section 3.2) states that keys used with HMAC-SHA 
algorithms MUST have a size >= 256 bits
```

**根本原因**：
JWT密钥 `hrms-dev-jwt-secret-key-2024` 只有28字节(224位)，不满足JWT规范要求的至少256位(32字节)。

**修复方案**：
更新 `application.yaml` 中的JWT密钥：

```yaml
# 修复前
jwt:
  secret: ${JWT_SECRET:hrms-dev-jwt-secret-key-2024}  # 28字节，不够

# 修复后
jwt:
  secret: ${JWT_SECRET:hrms-dev-jwt-secret-key-2024-must-be-32-bytes}  # 42字节，满足要求
```

**修复文件**：
- `backend/hrms-server/src/main/resources/application.yaml`

**修复状态**: ✅ 已修复

### 11. BCrypt密码Hash不匹配

**问题描述**：
登录时返回 `用户名或密码错误`，日志显示：
```
Encoded password does not look like BCrypt
```

**根本原因**：
数据库中的密码hash格式不正确，不是有效的BCrypt hash。

BCrypt hash格式要求：
- **前缀**: `$2a$` 或 `$2b$` 或 `$2y$`
- **Cost**: 2位数字 (通常10-12)
- **Hash**: 53个字符的base64编码
- **总长度**: 60个字符

**修复方案**：
更新数据库中的密码hash：

```sql
USE hrms;

-- 更新admin密码 (密码: password)
UPDATE sys_user
SET password = '$2a$10$dXJ3SW6G7P50lGmMkkmwe.20cQQubK3.HZWzG3YB1tlRy.fqvM/BG'
WHERE username = 'admin';

-- 验证密码长度
SELECT 
    username,
    real_name,
    LENGTH(password) as pwd_length,
    password
FROM sys_user 
WHERE username = 'admin';
```

**期望结果**: `pwd_length` 应该是 60

**最终登录凭证**：
- 用户名: `admin`
- 密码: `password`
- Token: 成功返回JWT token

**修复状态**: ✅ 已修复
