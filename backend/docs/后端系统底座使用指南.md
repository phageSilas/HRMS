# HRMS 后端系统底座使用指南

> 本指南面向项目开发成员，帮助快速理解和使用已实现的系统底座功能。

## 一、快速启动

### 1. 数据库准备

```sql
-- 1. 创建数据库
CREATE DATABASE hrms DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 2. 切换到 hrms 数据库
USE hrms;

-- 3. 执行表结构脚本（按顺序执行）
-- 路径：docs/sql/schema/
source docs/sql/schema/01_sys_user.sql;
source docs/sql/schema/02_sys_role.sql;
source docs/sql/schema/03_sys_menu.sql;
-- ... 其他表结构脚本

-- 4. 执行初始数据脚本
-- 路径：docs/sql/data/
source docs/sql/data/01_init_admin.sql;
source docs/sql/data/02_init_dict.sql;
```

### 2. 配置修改

修改 `backend/hrms-server/src/main/resources/application-dev.yaml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/hrms?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: your_username  # 修改为你的数据库用户名
    password: your_password  # 修改为你的数据库密码
```

### 3. 启动项目

```bash
# 方式 1：Maven 命令行
cd backend
mvn clean install
cd hrms-server
mvn spring-boot:run

# 方式 2：IDE 启动
# 在 IDEA 中直接运行 HrmsServerApplication.main()
```

### 4. 验证启动成功

访问健康检查接口：
```bash
curl http://localhost:8080/actuator/health
```

预期返回：
```json
{
  "status": "UP"
}
```

### 6. 操作日志

#### 使用注解记录日志

在 Controller 方法上添加 `@OperationLog` 注解：

```java
import com.hrms.system.log.annotation.OperationLog;

@RestController
@RequestMapping("/users")
public class UserController {

    @OperationLog(module = "用户管理", action = "创建用户")
    @PostMapping
    public Result<UserVO> create(@RequestBody UserRequestDTO request) {
        // 业务逻辑
    }

    @OperationLog(module = "用户管理", action = "更新用户")
    @PutMapping("/{id}")
    public Result<UserVO> update(@PathVariable Long id, @RequestBody UserRequestDTO request) {
        // 业务逻辑
    }

    @OperationLog(module = "用户管理", action = "删除用户", recordParams = false)
    @DeleteMapping("/{id}")
    public Result<Void> delete(@PathVariable Long id) {
        // 业务逻辑
    }
}
```

注解参数说明：
- `module`：操作模块（必填）
- `action`：操作类型（必填）
- `recordParams`：是否记录请求参数（默认 true）
- `recordResult`：是否记录响应结果（默认 false）
- `recordException`：是否记录异常信息（默认 true）

#### 查询操作日志
```bash
GET http://localhost:8080/operate-logs?module=用户管理&pageNum=1&pageSize=20
Authorization: Bearer {token}
```

## 三、开发规范

### 1. 统一返回体

所有接口使用 `Result<T>` 统一返回：

```java
// 成功
return Result.success(data);

// 失败
return Result.failure(ErrorCode.PARAM_ERROR);
return Result.failure(ErrorCode.USER_NOT_FOUND, "用户不存在");
```

响应格式：
```json
{
  "code": 0,
  "message": "success",
  "data": { ... }
}
```

### 2. 异常处理

使用业务异常：

```java
// 抛出业务异常
throw new BusinessException(ErrorCode.USER_NOT_FOUND);
throw new BusinessException(ErrorCode.PARAM_ERROR, "用户名不能为空");
```

常用错误码：
- `ErrorCode.SUCCESS` - 成功
- `ErrorCode.PARAM_ERROR` - 参数错误
- `ErrorCode.UNAUTHORIZED` - 未登录
- `ErrorCode.FORBIDDEN` - 无权限
- `ErrorCode.USER_NOT_FOUND` - 用户不存在
- `ErrorCode.AUTH_ERROR` - 认证失败

### 3. 使用实体类

所有实体类继承 `BaseEntity`：

```java
public class YourEntity extends BaseEntity {
    // BaseEntity 已包含：
    // - id (主键)
    // - createBy (创建人，自动填充)
    // - createTime (创建时间，自动填充)
    // - updateBy (更新人，自动填充)
    // - updateTime (更新时间，自动填充)
    // - isDeleted (逻辑删除标记)
}
```

### 4. DTO 和 VO

- **DTO**：接口入参（Data Transfer Object）
- **VO**：接口出参（View Object）

```java
// 入参 DTO
public class UserRequestDTO {
    private String username;
    private String password;
    // getter/setter
}

// 出参 VO
public class UserVO {
    private Long id;
    private String username;
    private String realName;
    // getter/setter
}
```

## 四、常见问题

### Q1：启动报错找不到表？

**A：** 检查数据库是否创建，表结构脚本是否执行。

### Q2：登录返回 401？

**A：** 检查：
1. 用户名密码是否正确
2. 用户是否被禁用（status = 0）
3. 数据库中是否有初始管理员账号

### Q3：接口返回 403？

**A：** 检查：
1. Token 是否正确传递（Authorization: Bearer {token}）
2. 用户是否有对应权限

### Q4：文件上传失败？

**A：** 检查：
1. 文件大小是否超过限制（默认 10MB）
2. 文件类型是否在允许列表中
3. 上传路径是否有写入权限

### Q5：日志没有记录？

**A：** 检查：
1. 方法上是否添加了 `@OperationLog` 注解
2. 日志模块是否正常启动
